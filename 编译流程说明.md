# 🚀 OpenWRT 云编译流程说明

## 总览

整个编译过程分为 **10 个步骤**，在 GitHub Actions 的 `ubuntu-22.04` 环境上运行。

---

## 第 1 步：🛎️ 检出代码 (Checkout)

**作用**：从本仓库拉取 CI 配置文件（脚本、平台配置、模板等）

**执行内容**：
```
actions/checkout@v4 → 检出仓库代码到 runner
```

**拉取内容**：
- `scripts/` — 编译脚本（9个）
- `config/` — 平台配置、插件定义、固件模板
- `.github/` — CI 工作流定义

---

## 第 2 步：📦 拉取源码 (Setup Source)

**作用**：克隆 OpenWrt 主源码 + 安装第三方软件包源

**执行脚本**：`scripts/setup-source.sh`

**流程**：
1. **克隆源码**：从 `https://github.com/LiBwrt/openwrt-6.x` 拉取 `main-nss` 分支（浅克隆 `--depth 1`）
2. **加载外来源**：读取 `config/feeds.yml` 中定义的第三方源
   - 默认包含：`kenzok8/openwrt-packages`、`kenzok8/small`
3. **更新 feeds**：`./scripts/feeds update -a`（拉取所有软件包索引）
4. **安装 feeds**：`./scripts/feeds install -a`（注册所有软件包）

**预计耗时**：3-8 分钟

---

## 第 3 步：⚙️ 生成配置 (Generate Config)

**作用**：根据用户选择的参数，组装 `.config` 编译配置文件

**执行脚本**：`scripts/generate-config.sh` + `scripts/apply-system-config.sh` + `scripts/validate-config.sh`

**流程**：
1. **查找平台配置**：在 `config/platforms/` 下查找目标平台的 `_platform.yml`
   - 当前目标：`qualcommax/ipq807x`
2. **组装 .config**（按顺序叠加）：
   - **目标平台**：`CONFIG_TARGET_qualcommax=y`
   - **设备型号**：默认全部设备（`CONFIG_TARGET_ALL_PROFILES=y`）
   - **基础模板**：`config/templates/base-firmware.config`（系统核心 + LuCI + 网络 + 存储 + USB + LED + 中文）
   - **NSS 加速**：`config/templates/nss.config`（高通硬件加速）
   - **防火墙模板**：`config/templates/firewall-iptables.config`
   - **用户插件**：通过 `--plugins` 参数指定
   - **ccache**：启用编译缓存加速
3. **应用系统配置**：设置 root 密码、LAN IP、WiFi SSID/密码
4. **运行 `make defconfig`**：自动补全依赖配置
5. **验证配置**：检查目标设备和防火墙类型是否正确

**预计耗时**：1-3 分钟

---

## 第 4 步：📋 配置摘要 (Config Summary)

**作用**：输出配置的可读摘要，方便确认

**执行脚本**：`scripts/config-summary.sh`

**输出内容**：
- 源码分支
- 目标平台 / 子目标
- 防火墙类型
- 已启用的插件数量
- 关键配置项

---

## 第 5 步：🏗️ 编译固件 (Build) ← 最耗时

**作用**：实际编译 OpenWRT 固件

**执行脚本**：`scripts/build.sh`

**流程**：
1. **📥 下载源码包**：`make download -j$(nproc)`
   - 下载所有软件包的源码压缩包
   - 并行下载失败时自动切换单线程重试
2. **🔧 编译工具链**：
   - `make tools/install -j$(nproc)` — 编译主机端工具
   - `make toolchain/install -j$(nproc)` — 编译交叉编译工具链（GCC、musl 等）
3. **🏗️ 编译固件**：`make -j${PARALLEL}`
   - 编译内核、所有软件包
   - 链接、打包固件镜像
   - 生成 `.bin` / `.itb` / `.img` / `.ubi` 文件

**并行数**：`CPU核心数 + 1`（GitHub runner 通常 2 核，并行数 = 3）

**预计耗时**：1-3 小时

**可能的错误**：
- 依赖缺失 → 检查 feeds 是否完整
- 编译错误 → 检查源码分支兼容性
- 磁盘空间不足 → GitHub runner 有 14GB 限制
- 超时 → 工作流设置了 360 分钟超时

---

## 第 6 步：🔍 编译后检查 (Post-Build Check)

**作用**：验证编译产物的完整性

**执行脚本**：`scripts/post-build-check.sh`

**检查内容**：
- 固件文件是否存在
- 文件大小是否合理
- 目标平台是否匹配

---

## 第 7 步：📊 构建摘要 (Build Summary)

**作用**：输出编译结果的统计摘要

**执行脚本**：`scripts/build-summary.sh`

**输出内容**：
- 编译耗时
- 固件文件列表及大小
- 磁盘使用情况

---

## 第 8 步：📄 生成清单 (Generate Manifest)

**作用**：生成 JSON 格式的构建清单

**执行脚本**：`scripts/generate-manifest.sh`

**输出**：`manifest.json` 包含：
- 构建参数（目标、分支、防火墙等）
- 固件文件列表
- 编译时间戳

---

## 第 9 步：📤 上传固件 (Upload Firmware)

**作用**：将编译产物上传为 GitHub Actions Artifact

**上传内容**：
- `*.bin` — 固件刷机包
- `*.itb` — FIT 镜像
- `*.img` — 磁盘镜像
- `*.ubi` — UBIFS 镜像
- `manifest.json` — 构建清单

**保留时间**：7 天

---

## 第 10 步：🧹 清理 (Post Checkout)

**作用**：GitHub Actions 自动清理临时文件
